1. What SnackFlow Does

SnackFlow tracks stock, handles two warehouses, manages shop sales through order bookers, records payments, tracks expenses, calculates profit, keeps GPS shop locations, and provides date-wise searchable reports.

2. User Roles

3. First Time Admin Setup

  1. Login as admin and change password.
  2. Verify Warehouse 1 and Warehouse 2.
  3. Create users and assign order bookers.
  4. Create products/SKUs and rates.
  5. Create shops, assign routes, and capture GPS.
  6. Enter opening stock and opening balances.

4. Product and SKU Management

Each product-size-flavour-pack combination is a separate SKU. Example: Chips Rs 50 BBQ, 24 packets per carton.

SnackFlow stores stock internally in packets, while screens can show cartons plus loose packets.

5. Warehouse and Inventory Management

Warehouse 1 and Warehouse 2 remain separate. Stock receiving converts cartons or bundles into packets and creates STOCK_IN ledger entries.

Never delete stock history. Use reversals, returns, or adjustment entries.

6. Shop Management and GPS

Go to Shops to add/edit shops. Record owner, phone, address, route, assigned warehouse, assigned order booker, credit limit, opening balance, and GPS coordinates. Use Open Map to verify Google Maps location.

7. Mobile Order Booker Workflow

  1. Open /mobile and login.
  2. Select assigned shop.
  3. Check pending balance.
  4. Add SKU, quantity, and rate.
  5. Review total bill and payment received.
  6. Save delivered order.
  7. Use payment-only or visit logging when needed.

8. Sales, Billing, Reversal, and Returns

Stock reduces only when a sale is delivered/confirmed. Booked orders do not reduce stock. Reversal corrects a full delivered sale. Partial return corrects selected returned items.

Previous Pending + Today Bill - Payment - Reversal/Return = New Pending

9. Payments, Expenses, and Profit

Payments reduce shop balance. Expenses are recorded date-wise. Gross Profit = Sales - COGS. Net Profit = Gross Profit - Expenses. Cash received is not the same as profit.

13. Security and Best Practices

  • Every user should have their own login.
  • Do not share admin password.
  • Change default password.
  • Reverse instead of deleting.
  • Backup database.
  • Review audit logs.

14. Glossary

SKU: Product + size + flavour + pack. COGS: Cost of goods sold. Reversal: Full correction. Partial return: Return selected sale items. Order booker: Field salesman.
